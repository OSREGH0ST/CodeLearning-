⚡ Event-Driven Systems

Non-Blocking Execution

💡 Performance Context

Never block the main thread. A single synchronous cryptographic operation, large JSON parse, or poorly optimized Regex can stall the entire Event Loop, dropping concurrent connections to zero. Offload CPU-bound tasks to Worker Threads.

Event Loop Phases (Node.js)

JavaScript is single-threaded, but the runtime is not. The Event Loop delegates I/O operations to the system kernel (via libuv). When an operation completes, the kernel tells Node.js to add the appropriate callback to the Poll Queue.

  • Microtasks: Executed immediately after the currently executing script and before any macroscopic task. (e.g., Promise.then, queueMicrotask).
  • Timers: Executes callbacks scheduled by setTimeout() and setInterval().
  • Poll: Retrieves new I/O events; executes I/O related callbacks (almost all with the exception of close callbacks, the ones scheduled by timers, and setImmediate()).

Promise Orchestration

Async / Await

Syntactic sugar over Promises that makes asynchronous code look synchronous. Always wrap await calls in try/catch blocks to handle rejected states, or use functional error handling wrappers.

Promise.allSettled()

Unlike Promise.all(), which short-circuits and rejects entirely if a single promise fails, allSettled waits for all promises to resolve or reject. Essential for batch processing where partial failures are acceptable.

Concurrent Execution Example
TypeScript 
async function fetchDashboardData(userId: string) {
    // Bad: Sequential blocking
    // const user = await getUser(userId);
    // const posts = await getPosts(userId);

    // Good: Concurrent execution
    const [userResult, postsResult] = await Promise.allSettled([
        getUser(userId),
        getPosts(userId)
    ]);

    // Handle fulfilled/rejected states individually...
    return { user: userResult, posts: postsResult };
}

Distributed Pub/Sub Systems

💡 Architectural Context

Event-driven microservices decouple producers from consumers. If the email service goes down, the checkout service shouldn't fail. It should drop an OrderPlaced event into the broker, and the email service will process it whenever it comes back online (Eventual Consistency).

Broker TypeTechnologyBest For...
Message Queue (Smart Broker) RabbitMQ, ActiveMQ Task routing, complex routing logic, and ensuring a message is processed exactly once by a specific worker (e.g., sending an email, processing a payment).
Event Stream (Dumb Broker) Apache Kafka, AWS Kinesis High-throughput log aggregation, telemetry, and event sourcing. Messages are persisted and can be replayed by multiple independent consumer groups.

Bidirectional Client Communication

WebSockets (WSS)

Upgrades a standard HTTP connection into a persistent, stateful, full-duplex TCP connection. Ideal for chat applications, live trading dashboards, and multiplayer games.

Server-Sent Events (SSE)

A unidirectional channel from server to client over standard HTTP. Much simpler to scale than WebSockets. Perfect for live news feeds or status updates where the client only needs to listen.

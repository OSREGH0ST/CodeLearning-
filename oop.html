🧱 Object-Oriented Programming

Los 4 Pilares de la POO

💡 Senior Dev Tip

La POO no es solo crear clases. Es sobre modelar el mundo real de forma que el código sea mantenible y extensible. Prioriza la composición sobre la herencia.

Encapsulamiento

Ocultar el estado interno de un objeto y solo permitir el acceso a través de métodos públicos.

Abstracción

Simplificar la complejidad ocultando los detalles de implementación y mostrando solo la funcionalidad necesaria.

Herencia

Permitir que una clase herede atributos y métodos de otra clase (Clase Padre).

Polimorfismo

La capacidad de un objeto de tomar muchas formas (ej. un método que se comporta diferente según el objeto que lo llama).

Principios SOLID

¿Por qué SOLID?

Son 5 principios de diseño destinados a hacer que los diseños de software sean más comprensibles, flexibles y mantenibles.

PrincipioDescripción
S: Single ResponsibilityUna clase debe tener una única razón para cambiar.
O: Open/ClosedAbierto para extensión, cerrado para modificación.
L: Liskov SubstitutionLas subclases deben ser sustituibles por sus clases base.
I: Interface SegregationMuchas interfaces específicas son mejores que una general.
D: Dependency InversionDepender de abstracciones, no de implementaciones.

Patrones Creacionales

Singleton

Garantiza que una clase tenga una única instancia y proporciona un punto de acceso global a ella.

Factory Method

Proporciona una interfaz para crear objetos en una superclase, pero permite que las subclases alteren el tipo de objetos que se crearán.
